Complete Flow Diagram - GradTrack

📋 Graduate Survey Journey

1 Graduate Receives Survey Link
Link shared via Email, SMS, Facebook, or QR Code
https://gradtrack.com/survey-verify?survey_id=1
2 Identity Verification Page
Graduate enters:
  • Student Number (e.g., 2020-12345)
  • Last Name (e.g., Doe)
  • Program (optional - BSCS, BSHM, etc.)
3 Backend Validation
System checks registrar database:
SELECT * FROM graduates
WHERE student_id = '2020-12345'
AND last_name LIKE '%Doe%'
AND program matches
✅ Found → Generate token
❌ Not found → Show error
4 Token Generation
System generates unique 64-character token:
Token: a7f3c9e2b8d4f1a6c5e9b2d8f4a1c7e3...
Expires: 30 minutes
Stored in: survey_tokens table
Token saved in sessionStorage (browser)
5 Survey Form Access
Graduate redirected to survey form
Welcome message: "Welcome, John Doe!"
Token validated on every page load
6 Survey Completion
Graduate answers all questions
Progress auto-saved to localStorage
Can navigate between sections
7 Survey Submission
System validates token again
Saves responses to database
Marks token as SUBMITTED
Records IP address and timestamp
8 Completion & Prevention
✅ Success message displayed
🔒 Token cleared from browser
🚫 Future submissions blocked
If graduate tries again: "Survey already submitted"

🛡️ Security Layers

Layer 1: Identity

Validates student number, last name, and program against registrar database. Only verified graduates can proceed.

Layer 2: Token

Generates unique 64-character cryptographic token. Expires after 30 minutes for security.

Layer 3: Duplicate Check

Database constraint prevents same graduate from submitting multiple times to same survey.

Layer 4: Audit Trail

Records IP address, timestamp, and submission status for every attempt. Full audit trail.

✅ System Working Correctly When:

  • Valid credentials redirect to survey form
  • Invalid credentials show error message
  • Survey displays graduate's name
  • After submission, cannot submit again
  • Token expires after 30 minutes
  • All submissions tracked in database

🔌 API Endpoints

POST /surveys/verify.php

Verifies graduate identity and generates token

POST /surveys/validate-token.php

Validates token before showing survey

POST /surveys/responses.php

Submits survey and marks token as used

GET /surveys/programs.php

Returns list of programs for dropdown
